# GridAligner 開発仕様書 (v1.2.1)

## 1. プロジェクト概要
**GridAligner** は、魚眼レンズや超広角レンズによる幾何学的な画像歪み（レンズ歪み・パース歪み）を、直感的な GUI 操作と AI 自動検出によって補正するためのデスクトップアプリケーションである。

> [!NOTE]
> 格子の自動検出は、**チェッカーボード（市松模様）等のテストパターン**、およびコントラストの明確な人工的な格子を主な対象とする。自然風景や一般的な建造物内の複雑な格子検出は、手動操作（コントロールポイントのドラッグ）による補正を前提とする。

## 2. コア機能の詳細

本ツールの機能は、[ARCHITECTURE_MANIFEST.md](../../ARCHITECTURE_MANIFEST.md) に定義された「レンズ補正先行フロー」および「絶対正規化座標系」の原則に基づき実装されている。

### 2.1 レンズ歪み補正 (Lens Correction)
* OpenCV の円形歪みモデルをベースとした補正エンジン。
* **k1 (第2次係数)**, **k2 (第4次係数)** のスライダー操作によるリアルタイム補正。
* バレル（樽型）およびピンクッション（糸巻き型）の両方に対応。
* **Intelligent Physical Sync**: レンズパラメータを変更した際、メッシュの各点を「旧レンズ空間 → 元画像空間 → 新レンズ空間」へと再投影（Reproject）する。これにより、グリッドが画像内の物理的な特徴（交点等）を自動的に追いかけ、補正の前後での整合性を維持する。

### 2.2 メッシュ・レクティファイ (Mesh Rectification)
* 自由度の高い格子状メッシュによる局所的な歪み（パース・台形歪み）の補正。
* コントロールポイント（赤丸）をマウスでドラッグし、画像内の格子線に合わせることで直線を復元。
* **Smooth Warp (Physics)**: 物理演算ベースのエネルギー最小化により、数点の補正から周辺のメッシュを滑らかに追従させ、格子の連続性を維持する。

### 2.3 AI 格子自動検出 (Auto-Grid Detection V2)
チェッカーボード認識と投影プロファイル解析を統合したハイブリッドエンジン。
（技術的な採用根拠については [ARCHITECTURE_MANIFEST.md](../../ARCHITECTURE_MANIFEST.md) を参照）

*   **ハイブリッド認識の構成**:
    *   **チェッカーボード・フィッティング**: `cv2.findChessboardCornersSB` 等を用いた高精度認識。
    *   **投影プロファイル解析 (Fallback)**: エッジ密度解析による統計的ピーク判定。
    *   **色依存なし (Color-Agnostic)**: RGB 各チャンネルからエッジを抽出・合成。
    *   **コーナー・サブピクセル吸着**: `cv2.cornerSubPix` による 1ピクセル以下の精度でのスナップ。

*   **格子検出の精度向上の方針**:
    *   **SB (Sector Based) 方式**: 現在採用している方式により、角度の付いたボードでも安定して検出可能。
    *   **再検討のトリガー**: 将来的に、極限まで「うねり」を排除した超高精度な幾何補正が要求される場合には、Meta（旧Facebook）が提唱する Deltille Grids 系のアルゴリズム導入を検討する。
    *   **参考**: [Deltille Detector (GitHub)](https://github.com/deltille/detector) / [OpenCV実装より高精度な検出器 (Qiita)](https://qiita.com/fukushima1981/items/0e0f2a56b115d3271ebf)

### 2.4 アドバンスド・ユーティリティの詳細仕様

*   **Grid Straighten (格子整列)**:
    *   **目的**: キャリブレーションシートの「物理的な紙のしなり」による不要な歪みを、数学的にキャンセルして理想的な平面を復元する。
    *   **仕組み**: 四隅を基準とした射影変換（ホモグラフィ）に基づき、内部の全交点を幾何学的に正しい直線上へ強制配置する。
*   **Grid Extrapolation (格子外挿)**:
    *   **目的**: 壁に貼った小さなボード一枚を AI 検出するだけで、その平面（パース）を画像全体に拡張し、画面全体の垂直・水平を正す。
    *   **仕組み**: 部分的なメッシュからホモグラフィを算出し、画像全体の四隅をその平面上に逆投影（外挿）する。
*   **Grid Orientation Correction (格子向き補正)**:
    *   **目的**: AI が格子の「左上」を誤認した場合、出力前に手動で向きを正し、FFmpeg 等の外部ツール出力時の映像反転を防ぐ。
    *   **動作**: 格子配列を90度回転し、長方形格子の場合は `Rows/Cols` 数値を自動スワップする。

### 2.5 描画補助機能 (Visual Aids)
*   **Crosshair Mode**: マウスクリックした地点を通る「水平・垂直の十字ガイド」を表示。
    *   **役割**: 全面の格子表示が画像細部を隠してしまう場合でも、特定のライン（水平線、柱等）が完全に真っ直ぐになったかを「バーチャル定規」として確認できる。

## 3. ワークフローとパイプライン

1.  **レンズ補正フェーズ**: 放射歪みの除去。AI が「直線」を探しやすくする。
2.  **メッシュ変形フェーズ**: 補正済み画像に対するホモグラフィ/局所変形の適用。
3.  **プレビュー生成**: PyTorch Tensor 演算（`grid_sample`）による高品質なリアルタイム描画。

> [!TIP]
> **同期機能の恩恵**: 物理座標同期機能（Intelligent Physical Sync）により、「レンズ補正」と「メッシュ変形」の調整順序は任意となった。先にメッシュを合わせた後にレンズ歪みを調整しても、メッシュが自動追従するため、試行錯誤が極めて容易になっている。
